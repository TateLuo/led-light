# Battery SOC Test Table

用途：记录外壳已组装状态下的电池电压、电量显示、温度和负载变化，用于校准 2S 电池电量估算算法。

测试建议：

- 测试前电池静置 10 分钟。
- 优先在电池电压 7.6V 到 8.2V 区间测试。
- 低于 6.6V 时不要长时间满载测试。
- 每个测试段开始前，让 LED 关闭 30 秒作为基准。
- 如果能接串口，优先记录日志里的电压；如果不能，就用手机录像拍屏幕，测试后按时间点填写。

## 固件记录器操作

1. 长按 SW2 进入设置页。
2. 在 Battery Test 项向右旋转 SW1 开始自动测试；显示 RUN 和当前样本数。
3. 运行中再次向右旋转 SW1 会暂停并安全关灯，显示 PAUSE；再次向右旋转继续。
4. 向左旋转 SW1 可随时停止，显示 STOP；全部步骤完成后显示 DONE。
5. 自动测试约 17 分 20 秒，依次执行关灯基准、暖光满载、冷光满载、混光满载、
   中等负载、三次负载突变和手动风扇影响测试，每秒记录一条数据。
6. 测试 STOP 或 DONE 后，在 WiFi Service 项启动热点，连接后打开 192.168.4.1，
   点击 Download CSV。
7. CSV 成功发送完成后，设备自动清空本次测试数据。

记录保存在 RAM 中，关机或掉电后会丢失。运行或暂停时禁止下载。容量为 1800 条，
约 30 分钟；超过容量后会覆盖最早的数据，并在 CSV 的 sequence 列体现。

自动测试不会绕过安全管理器。严重低电、过温、NTC 或 ADC 故障会立即停止测试、
关闭 LED 并保留已经记录的数据。

自动测试与 OTA 互锁：测试运行或暂停时不能启动 WiFi Service，Wi-Fi 服务活动时
也不能开始测试。只要仍有测试数据，固件上传接口就会拒绝 OTA 写入；必须先成功
下载 CSV 并由设备自动清空数据，才能上传新固件。
